# Security Policy

## Supported versions

This project is in an early `0.x` stage. Security fixes, when available, are expected to land on the latest development line rather than being backported to multiple release branches.

| Version | Supported |
| ------- | --------- |
| 0.x     | :white_check_mark: |
| < 0.1   | :x: |

## Reporting a vulnerability

Please do **not** open a public GitHub issue for suspected vulnerabilities.

Preferred reporting path:

1. Use GitHub's private vulnerability reporting or repository security advisory flow if it is enabled.
2. If private reporting is not yet enabled, contact the maintainers through a private channel on the repository hosting platform before public disclosure.

Please include:

- a clear description of the issue
- affected files, APIs, or flows
- impact assessment
- reproduction steps or a proof of concept, if safe to share
- suggested remediation, if you have one

## Response expectations

Because this project is still early-stage, response times may vary. Maintainers will try to:

- acknowledge receipt of a report in a reasonable time
- validate the issue
- coordinate a fix before public disclosure when appropriate

## Hardening notes

At the time this file was added, the repository did not yet include a dedicated security contact address. Add one before wide external adoption if you want a stronger disclosure path.
